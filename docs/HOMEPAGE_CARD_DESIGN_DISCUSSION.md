# 首页卡片设计讨论 - 标题字段优化

> 产品经理视角：分析标题字段的必要性和优化方案

---

## 📊 当前卡片结构分析

### 当前信息层级

```
┌─────────────────────────────────┐
│ [类型标签] [时间]                │  ← Header（必要）
├─────────────────────────────────┤
│ 标题（headline）                │  ← 17px，最多15字
│ 竞品A发布低价套餐                │
│                                 │
│ 副标题（subheadline）            │  ← 13px，描述发生了什么
│ 过去 24 小时发布 Pro Lite 套餐  │
│ + 渠道联合促销，目标中型客户。   │
├─────────────────────────────────┤
│ STRATEGY                        │
│ AI建议内容...                    │  ← AI建议（核心价值）
└─────────────────────────────────┘
```

### 问题分析

#### 1. 信息重复
- **标题**：`"竞品A发布低价套餐"`（15字限制）
- **副标题**：`"过去 24 小时发布 Pro Lite 套餐 + 渠道联合促销，目标中型客户。"`（完整描述）

**结论**：副标题已经包含了标题的信息，标题只是简化版

#### 2. 空间浪费
- **标题**：17px 字号，占用 1-2 行空间
- **副标题**：13px 字号，占用 2 行空间（line-clamp-2）
- **总占用**：约 3-4 行空间

**结论**：标题占用了宝贵的垂直空间

#### 3. 价值有限
- **标题的作用**：
  - ✅ 快速扫描？→ 但副标题已经说明了
  - ✅ 视觉层次？→ 但可能不够明显
  - ✅ 信息密度？→ 确实降低了

**结论**：标题的价值有限，可能不如直接展示副标题

---

## 🎯 产品设计原则

### 1. 信息密度原则
- **目标**：在有限空间内展示最大价值信息
- **当前**：标题 + 副标题 = 信息重复
- **优化**：移除标题，直接展示副标题

### 2. 认知负荷原则
- **目标**：减少用户理解成本
- **当前**：需要阅读标题 + 副标题（重复信息）
- **优化**：只读副标题（完整信息）

### 3. 视觉层次原则
- **目标**：清晰的视觉层次
- **当前**：标题（17px）+ 副标题（13px）
- **优化**：副标题提升为主标题（16-17px）

## 💡 优化方案

### 方案1：移除标题，副标题提升为主标题（推荐）

#### 设计
```
┌─────────────────────────────────┐
│ [类型标签] [时间]                │
├─────────────────────────────────┤
│ 过去 24 小时发布 Pro Lite 套餐  │  ← 16-17px，主标题
│ + 渠道联合促销，目标中型客户。   │
│                                 │
│ STRATEGY                        │
│ AI建议内容...                    │
└─────────────────────────────────┘
```

#### 优势
- ✅ **信息密度提升**：移除重复信息
- ✅ **空间节省**：节省 1-2 行空间
- ✅ **认知负荷降低**：只需阅读一次
- ✅ **视觉更清晰**：副标题提升为主标题

#### 劣势
- ⚠️ **标题可能更长**：但可以 line-clamp-2 控制
- ⚠️ **快速扫描可能变慢**：但完整信息更有价值

---

### 方案2：标题改为关键词标签

#### 设计
```
┌─────────────────────────────────┐
│ [类型标签] [时间]                │
├─────────────────────────────────┤
│ [竞品A] [价格调整] [中端市场]    │  ← 关键词标签（可选）
│                                 │
│ 过去 24 小时发布 Pro Lite 套餐  │  ← 主标题
│ + 渠道联合促销，目标中型客户。   │
│                                 │
│ STRATEGY                        │
│ AI建议内容...                    │
└─────────────────────────────────┘
```

#### 优势
- ✅ **快速扫描**：关键词一目了然
- ✅ **信息补充**：标签提供额外维度
- ✅ **视觉层次**：标签 + 标题 + 建议

#### 劣势
- ⚠️ **空间占用**：标签占用额外空间
- ⚠️ **复杂度增加**：需要 AI 提取关键词

---

### 方案3：标题改为紧急度/影响度标识

#### 设计
```
┌─────────────────────────────────┐
│ [类型标签] [时间]                │
├─────────────────────────────────┤
│ [高紧急] [高影响]                │  ← 紧急度/影响度（可选）
│                                 │
│ 过去 24 小时发布 Pro Lite 套餐  │  ← 主标题
│ + 渠道联合促销，目标中型客户。   │
│                                 │
│ STRATEGY                        │
│ AI建议内容...                    │
└─────────────────────────────────┘
```

#### 优势
- ✅ **优先级清晰**：快速识别重要事件
- ✅ **信息补充**：提供决策维度

#### 劣势
- ⚠️ **空间占用**：占用额外空间
- ⚠️ **可能冗余**：类型标签已经表达了

---

## 🎨 推荐方案：方案1（移除标题）

### 理由

1. **信息密度最大化**
   - 移除重复信息
   - 节省空间
   - 提升信息价值

2. **认知负荷最小化**
   - 只需阅读一次
   - 完整信息直接展示
   - 减少理解成本

3. **视觉层次清晰**
   - 副标题提升为主标题（16-17px）
   - 保持清晰的视觉层次
   - 符合设计系统

4. **AI 友好**
   - AI 只需生成副标题（完整描述）
   - 不需要生成简化标题
   - 降低 AI 生成复杂度

### 实施细节

#### 1. 副标题提升为主标题
```typescript
// 当前
<h3>{headline}</h3>           // 17px
<p>{subheadline}</p>          // 13px

// 优化后
<h3>{subheadline}</h3>        // 16-17px，主标题
// headline 字段移除或忽略
```

#### 2. 行数控制
```typescript
// 使用 line-clamp-2 或 line-clamp-3
<p className="line-clamp-2">{subheadline}</p>
```

#### 3. 数据源调整
```typescript
// AI 生成时，只需要生成 subheadline
// headline 可以移除或忽略
```

## 🎯 最终建议

### 推荐：方案1（移除标题，副标题提升为主标题）

**理由**：
1. ✅ **信息密度最大化**：移除重复信息
2. ✅ **空间节省**：节省 1-2 行空间
3. ✅ **认知负荷降低**：只需阅读一次
4. ✅ **AI 友好**：AI 只需生成完整描述
5. ✅ **实施简单**：只需调整组件，不需要数据源改动

**实施步骤**：
1. 移除 `headline` 字段的显示
2. 将 `subheadline` 提升为主标题（16-17px）
3. 使用 `line-clamp-2` 或 `line-clamp-3` 控制行数
4. 保持其他设计不变

## 💬 讨论点

### Q1: 如果移除标题，快速扫描会不会变慢？
**A**: 
- 副标题提升为主标题后，视觉层次仍然清晰
- 完整信息更有价值，减少二次理解
- 可以通过类型标签快速筛选

### Q2: 标题是否可以改为其他用途？
**A**: 
- 可以改为关键词标签（方案2）
- 可以改为紧急度标识（方案3）
- 但会增加复杂度，需要评估价值

### Q3: 如果保留标题，如何优化？
**A**: 
- 标题改为更抽象的描述（如"价格策略调整"）
- 副标题提供具体细节
- 但这样可能增加理解成本

---

## 📝 总结

**当前问题**：
- 标题和副标题信息重复
- 标题占用宝贵空间
- 标题价值有限

**推荐方案**：
- 移除标题字段
- 副标题提升为主标题（16-17px）
- 使用 line-clamp 控制行数

**预期效果**：
- 信息密度提升 30-40%
- 空间节省 1-2 行
- 认知负荷降低
- AI 生成更简单
